# Design Token 与主题（治配色不统一）

全局只锁一套 design token，全程不跑偏。来源：huashu 品牌资产协议（§1.a）+ guizang 主题色规则（themes-swiss）+ 方法论15 #56。

目录：

## 品牌资产协议（先定 token，再画任何一页）—— 借 huashu §1.a

设计开始前，先把这套 token 锁死（写进 deck 项目的 design token，全 deck 引用）：

1. **问客户要 VI**：logo / 品牌主色 / 字体规范。
2. **搜官方渠道**：官网、品牌手册，grep 出 `#xxxxxx` 色值（按频率排序，过滤黑白灰）。
3. **定一套 token**：`{accent, ink, paper, font_sans, grid:12}`。客户有品牌色 → 用客户主色当 accent（方法论15 #56：主色调与客户 VI 一致是尊重）；没有 → 从 guizang 4 套瑞士主题选一套。
4. **绝不从记忆猜品牌色**（huashu 铁律）——必须来自客户 VI 或 guizang 预设。

> token 一旦锁定，写进 deck 项目记录，后续迭代保持一致。

## guizang 4 套瑞士主题（无客户 VI 时的备选）

每套 = 高级灰白底 + **单一**高饱和高亮色。详细色值见项目 `templates/guizang-refs/themes-swiss.md`：

| 主题 | accent | 适合 |
|---|---|---|
| 🔵 克莱因蓝 IKB | `#002FA7` | 通用/商业/科技/设计（不知道选啥就这个）|

## 单一强调色铁律（治"橙色突然变蓝"）

- **一份 deck 只用一套主题，全程不中途换 accent**（guizang themes-swiss 硬规则）。
- **禁止混搭**两个高亮色（IKB 蓝 + 柠檬黄同时作高亮 = 彻底违反瑞士风"单一锚点色"）。
- 不允许用户自定义任意 hex（委婉拒绝，展示预设让选）。
- accent 不要泛滥——KPI 数字/单字符高亮用即可，满屏 accent 会掉档。
- `audit_visual.py` 会扫全 deck 的强调色种类，多于一个直接 FAIL。

## 灰阶（4 套主题完全相同，不要改）

`--paper:#fafaf8`（极浅暖白）· `--grey-1:#f0f0ee` · `--grey-2:#d4d4d2`（分割线）· `--grey-3:#737373`（辅助文字）· `--ink:#0a0a0a`（近黑）。

不要改成纯白 `#fff` 或纯黑 `#000`——会损失瑞士风的克制质感。
