# SCQA 开场 + 金字塔骨架

把"开头讲清要解决什么问题"和"观点先行、自上而下推导"两件事做对。来源：Barbara Minto 金字塔原理 / MBB 通用 SCQA + 方法论15（#1/#2/#84/#45/#100）。

目录：
- SCQA：方案怎么开场
- 金字塔：怎么搭骨架
- 从结尾倒推
- 可记忆抓手

## SCQA：方案怎么开场（对应 brief_opening）

开场不是介绍，是把读者带进"必须回答的那个问题"。四段，各一两句：

- **S 现状 (Situation)**：客户当前所处的客观局面，对方认可的事实。从这里开始，让读者点头。
- **C 冲突 (Complication)**：出现了什么变化/矛盾/威胁，让现状不可持续。这是张力的来源。
- **Q 问题 (Question)**：因此必须回答的那**一个**核心问题——它就是 brief 的根问题。
- **A 答案 (Answer)**：你的顶层主张。先亮出来（金字塔顶），后面所有章节为它服务。

会看会想 brief（方法论15 #1/#2）：brief 上写的不一定是真问题。先提炼出最精炼的核心问题，再写 Q。"提出一个好问题，等于答案的一半。"

> SCQA 的 S+C+Q 落进 `brief_opening`；A 落进 `conclusion.governing_thought`，并在开场就预告。

## 金字塔：怎么搭骨架

自上而下，先结论后支撑：

```
            顶层结论 A（一句话主张）
           /        |         \
       论点1       论点2       论点3      ← 每个论点 = 一个章节
      / | \       / | \       / | \
    论据 论据    论据 论据    论据 论据     ← 每条论据 = 一页(或几页)
```

- **顶层结论**：就是 SCQA 的 A。
- **论点（章节）**：3-5 个，每个是支撑 A 成立的一个理由。
- **论据（页）**：每个论点下的事实/分析卡。

自检（金字塔自洽）：把各章标题**连起来读**，应该能独立讲通"为什么 A 成立"。读不通 → 章节切错了，重搭。

每页标题写成**判断句**（#100 每页写核心句）：标题就是这页的结论，内容是支撑它的论据。读者扫标题即可获得论证主线。

## 从结尾倒推（方法论15 #84）

动笔前先回答：**方案读完，你最希望客户记住的那一句话是什么？** 那句话就是 A。然后让所有章节、每一页都为这句话服务。与命题无关的内容，再好也砍掉（#32/#80 能删则删）。

## 可记忆抓手（方法论15 #45）

提案结束客户会忘记 90%。设计一个"可记忆的抓手"——核心策略名（如"大闭环+小闭环"）、一句金句、或一个生动比喻——并**全篇重复 3 次以上**（开场预告、章节呼应、结尾 callback）。这与叙事的"回环呼应"是同一件事。
