# 找可用结论（竞品分析的目的）

分析只是手段，结论才是目的。绝大部分竞品分析陷入"有分析无结论"的怪圈——讲了一堆竞品做了什么，却没一句"所以我们该怎么办"。来源：方法论03 §2.3 + market-research 决策导向。

目录：
- 结论三问
- 决策导向：区分事实/推断/推荐
- 结论的质量标准
- battle card（可选交付形态）

---

## 结论三问（每个拆解都要收敛到这三问）

1. **借鉴什么？** 它做对的、值得我们学的。
2. **规避什么？** 它的坑、我们不能踩的。
3. **基于自身情况，我们接下来该怎么做？** 这一条是落点，写进分析卡的 `implication`。

没有第 3 问，就是无效分析。

---

## 决策导向：区分事实 / 推断 / 推荐（market-research 实证）

写结论时分清三层，不要混在一起：
- **事实**：竞品客观做了什么（可溯源）。
- **推断**：基于事实你的判断（标注为判断，可能错）。
- **推荐**：我们该怎么做（行动）。

分析卡里：`evidence` = 事实，`claim` = 推断/判断，`implication` = 推荐/行动。三者一一对应、不越位。

---

## 结论的质量标准

好结论要：**有策略性**（不是泛泛而谈）、**极简不复杂**、**能引导执行**、**能落地**。

反例（不合格）：信息很多、结论很少，或结论只是"总结表面现象"而非深刻洞察。
正例：一句话点出"决定成败的根本变量"，并给出我们相对它的差异化动作。

## battle card（可选交付形态）—— battlecard ≠ feature matrix

把对核心竞品的结论压缩成一张"对战卡"。关键提醒（真实 battlecard skill 实证：Hashmeta / Arcade / termo）：**只把竞品功能并排列出来的是 feature matrix，不是 battlecard，帮不了赢单**。真正的 battlecard 有四块：

1. **竞品快照（Snapshot）**：它是谁、定位、主攻市场、典型客户画像——让人秒认"什么场景该掏这张卡"。
2. **诚实列出竞品强项**：先承认对方做得好的，建立可信度，也让对话时不露怯。
3. **我方差异化**：相对它我们赢在哪（对应 self-analysis 的真优势）。
4. **应对话术**：客户拿它对比时，一句话怎么回。

为速度设计：**2 分钟内能扫完、对话中能随时调用**，而不是写成没人读的长报告。适合销售赋能或竞争激烈的方案，非必须。

> 另一个可视化工件：**positioning map（感知地图）**——把竞品摆到两条关键轴上，直观露出无人占据的空位，供叙事层做对比页。
