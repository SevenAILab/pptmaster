# 三层拆解 + 四场景框架

拆解不只看对方做了什么，更要看"为何这样做"。只看表面动作，理解的只是皮毛。来源：方法论03 §2.2 + §3 + Porter 战略意图 + review mining。

目录：
- 三层拆解（事实/动机/效果）
- 找决定成败的根本变量
- 四场景专属框架

---

## 三层拆解

### 事实层：做什么？
具体动作是什么？时间节奏怎么铺的？资源如何分配？
（看产品真相与实际动作，不抄它的营销文案 —— market-research 实证）

### 动机层：为何做？
项目背景是什么？跟行业/企业的相关性？为何在这个节点做？目标是什么、要什么结果？策略是什么、用什么方法解决问题？

> Porter 视角强化：透过动作看**战略意图**——它在下哪盘棋？防守哪块、进攻哪块？这比罗列动作深一层。

### 效果层：做得如何？
拿到什么结果？数据表现与认知转变？成功的关键因素 / 失败的核心原因？

> review mining 强化：效果不只看官方数据，去社媒/电商/社区挖**真实用户口碑**——被夸最多的优点、被骂最多的槽点、用户实际用法与官方定位的偏差。一句真实用户原话，常比冰冷数据更有说服力。

---

## 找决定成败的根本变量

拆到最后，要找到那个**决定项目成败的根本变量**——它决定了事情的走向。找到主要矛盾，就找到了本质。分析卡的 `claim` 应该落在这个根本变量上，而不是停在表面动作。

## 四场景专属框架（按本方案的场景选用）

### 场景 1 · 营销传播：背景 → 目标 → 策略 → 创意 → 执行 → 效果
分析顺序：先看背景和目标（为何做、想要什么）→ 再看策略和创意（方向对不对、表达妙不妙）→ 然后执行（落地效率、资源合理性）→ 最后效果（值不值得学）。效果看四类指标：曝光 / 互动 / 转化 / 效率。

### 场景 2 · 产品定位：用户 → 场景 → 问题 → 解决方案 → 支撑点
- 用户：画像 / 消费偏好 / 潜在动因（最深一层，常是焦虑、归属、自我实现等心理需求）。
- 场景：生活 / 情绪 / 社交场景；判断高频刚需 / 低频高价值 / 新场景。
- 问题：痛点（不解决不行）/ 痒点（影响体验）/ 爽点（满足后离不开）。
- 解决方案：功能层 / 情绪层 / 社交层。
- 支撑点(RTB)：技术证明 / 信任背书 / 口碑效应 / 设计细节 / 品牌势能。
- 一句话定位公式：为【目标用户】，在【使用场景】下，解决【某类问题】的【产品品类】。

### 场景 3 · 产品上市（0-1 / 1-10）
上市节奏（慢热/爆破/长尾）/ 预算分配 / 平台组合（各渠道角色）/ 内容策略（怎么讲产品、用什么概念）/ 投放动作 / 商业结果。看清别人为何成功或失败，再想自己怎么做得更好。

### 场景 4 · 品牌升级
过去什么样（哪里不行）→ 面临什么问题（为什么改）→ 升级成什么样（新方向）→ 做了哪些事（定位/VI/产品/传播）。三大典型：年轻化 / 高端化 / 多元化。

> 选框架原则：方案是什么类型，就用对应场景框架拆竞品；不要四个都套。
