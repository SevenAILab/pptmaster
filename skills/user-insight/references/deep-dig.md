# 挖深度（行为 → 场景 → 动机）

九字诀第三步：判断真假后，深挖真需求。六字诀：行为→场景→动机。来源：方法论05 §3 + JTBD 框架。

目录：
- 看行为
- 还原场景
- 找动机（5Why）
- JTBD 三层

---

## 看行为（不看说，看做）

不要看用户说什么，要看他们做什么——行为是最好的证明。三看：
- **看替代**：现在用什么替代方案？替代越差/越贵/越麻烦，机会越大；没替代说明不够痛。（网约车切入出租车的诸多痛点）
- **看卡点**：在哪个环节卡住了？卡点=痛点=切入点。（美图秀秀对 PS：一键美颜，去掉修图门槛）
- **看冲突**：什么行为看上去矛盾？不合常理的行为往往藏更深洞察。（微信读书：划线多的人爱分享金句——阅读是社交货币；B站：弹幕挡脸还反复刷——要的是群体共鸣）

---

## 还原场景（五要素）

脱离场景谈需求都是隔靴搔痒。场景五要素：**谁 + 在哪 + 在什么状态下 + 遇到什么问题 + 当时情绪如何**。

同样喝咖啡：办公室提神（功能需求→瑞幸快取）vs 约朋友聊天（社交需求＞提神→星巴克第三空间）。用户没变，场景变了，需求就变了，产品定义逻辑也不同。

---

## 找动机（5Why 挖底层）

用户最直接的回答往往是表面需求，第一层想到的还没到本质。用 5 个 Why 连续追问，挖到真正驱动行为的底层动机。

例（减肥）：看起来油腻 → 在意同事聚会形象 → 被说中年发福 → 对状态不满想找回自信 → **底层是掌控感**。表面需求是减肥，底层动机是掌控感与自信（Keep "自律给我自由" 抓的就是这个）。

## JTBD 三层（功能 / 情感 / 社交）

借 Jobs-to-be-Done 把"用户雇佣产品完成什么任务"拆三层，与上面的动机挖掘对齐：
- **功能性 job**：要完成的实际任务（从 A 到 B）。
- **情感性 job**：想获得的感受（安心、掌控、被认可）。
- **社交性 job**：想在他人面前塑造的形象（社交货币）。

深挖时三层都问，洞察往往藏在情感/社交层。
