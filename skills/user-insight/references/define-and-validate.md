# 定用户 + 判真假

用户分析九字诀的前两步：定用户、判真假。来源：方法论05 §1-2。

目录：
- 定用户（三问法）
- 判真假（三个够）

---

## 定用户（三问法）

战略就是聚焦和取舍——选择服务谁，就决定了不服务谁。定义目标用户要敢做减法：每多覆盖一类人群，产品定义就多妥协一层。宁可在特定人群把需求做到极致，也不要在泛人群做到平庸。

1. **他们是谁？** 不是泛泛的性别/年龄/城市/收入标签，而是从问题出发：谁对这个问题痛感最强、付费意愿最高？先从痛感最强、最愿尝鲜的**种子用户**下手，再向外延展。
2. **他们在哪里？** 聚集在什么平台/渠道/圈子？能否**低成本触达**？找到了但触达成本极高，也不是好生意。
3. **他们现在用什么方案解决？** 已在用某种（不够好的）方案 = 需求存在；什么都不做 = 可能不够痛。**现有方案的缺陷，就是你的切入点。**

## 判真假（三个够：够真 × 够多 × 够落地）

很多人把细分做得很细、标签做得很漂亮，但若本身是伪需求，就是千斤拨四两。判断真伪不是问用户，而是看行为、看场景、看支付意愿。

### 够真（需求强度）：痛点 / 痒点 / 爽点
- **痛点**：不解决不行的刚需（网约车解决打车难）——付费核心驱动。
- **痒点**：有了明显提升体验（拍照从能用到好用）——消费升级驱动。
- **爽点**：超预期、一旦体验回不去（系统丝滑）——溢价/复购/忠诚驱动。
- 反判：痛点不够痛、痒点不够痒、爽点不够爽，三者都不占 = 需求不够真。"叫好不叫座"的本质是优先级太低。

### 够多（市场规模三看）
- 现有市场花了多少钱？（客单 × 频次能撑多大盘）
- 从需求到付费要过几道坎？（漏斗：有问题→想解决→愿花钱→买你，每道坎筛掉一批，如左撇子菜刀）
- 供应链能不能跟上？（市场太小，供应链没理由配合）

### 够落地（交付三问）
- 需求能解决到什么程度？（用户期待 10 分你能做几分；差距 <20% 边做边迭代，>50% 先 MVP 或等契机）
- 成本有没有优势？（早期生鲜电商：需求真但冷链履约跟不上）
- 时机对不对？（外卖 2014 才爆发=智能机+移动支付+众包物流三条件同时成熟）
