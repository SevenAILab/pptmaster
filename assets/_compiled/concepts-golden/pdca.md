# PDCA 循环 · Plan-Do-Check-Action

## 定义

PDCA 是 Plan、Do、Check、Action 四步循环,中文常称戴明环,用于把计划、执行、检查和改进行动连接成持续优化闭环。

模型卡中将 PDCA 定义为全面质量管理遵循的科学程序: P 是计划目标,D 是执行内容,C 是检查结果并找出问题,A 是处理检查结果,将成功经验标准化,将失败教训规避,未解决问题进入下一轮循环。Seven 品牌传播 SOP 中也强调,每个传播阶段完成后都需要反思和检查,找到优点和不足,让品牌塑造更符合企业、市场和发展需要。

在 PPTAgent 中,PDCA 是年度规划和执行复盘的底层方法,确保品牌方案不是一次性交付,而能被追踪、评估和迭代。

## 适用场景

- **年度规划执行闭环**: Sub-Agent ⑥ 将 OKR、营销日历和 Campaign 纳入复盘节奏
- **IMC 传播优化**: 计划核心信息和渠道,执行内容,检查效果,优化下一轮
- **品牌资产管理**: 定期检查视觉、语言、内容和触点是否一致
- **竞品动态跟踪**: 用固定周期更新竞品矩阵和感知地图
- **不适合**: 目标不清或指标不可衡量的项目；没有 Check,PDCA 会退化为 "做了就算"

## 使用步骤

### 第 1 步 · Plan 计划

明确目标、假设、指标、资源、时间和负责人。品牌项目中要把目标写成可检查的指标,例如品牌搜索增长、试用转化、复购率、内容收藏率。

输出标准: 计划表 + KPI + 风险假设。

### 第 2 步 · Do 执行

按计划执行,同时记录过程数据和偏差。执行阶段不要只看结果,也要记录哪些素材、渠道、话术、合作方和时间点产生了影响。

输出标准: 执行记录 + 素材清单 + 异常记录。

### 第 3 步 · Check 检查

对照目标检查结果,用 O Data 和 X Data 一起看: 运营数据看发生了什么,体验数据看为什么发生。

输出标准: 结果对比 + 关键发现 + 根因判断。

### 第 4 步 · Action 行动

把有效做法标准化,把失败教训转成禁区或改进动作,未解决问题进入下一轮计划。

输出标准: 保留/停止/优化/新增清单。

### 第 5 步 · 沉淀为 SOP

将可复用经验沉淀到 Brief、内容规范、渠道规范、视觉规范和复盘模板中。

输出标准: 更新后的 SOP 或 checklist。

### 第 6 步 · 开启下一轮循环

把 Action 结论转入下一周期 Plan,避免复盘只停留在会议纪要。

输出标准: 下一轮计划输入 + 负责人 + 截止时间。

## 输入输出示例

### 示例 1 · 植愈坊小红书种草 PDCA (247 书贯穿教学案例)

> **案例可追溯报告**: 植愈坊是 Seven 247 书全书贯穿的**教学性虚拟案例** (非真实商业体)。具体出处行号待 Codex 后续按 P0-4 模板填充, 当前仅标注教学属性。**禁止包装成"真实品牌成功案例"**。

背景: 植愈坊计划验证 "每日芳香疗愈仪式" 是否能带来小红书种草和试用转化。

| 阶段 | 输出 |
|---|---|
| Plan | 4 周发布 20 篇 KOC 笔记,核心信息为自我疗愈仪式,目标收藏率 8%、试用 100 单 |
| Do | 执行芳疗知识、浴室仪式、敏感肌体验、开箱审美 4 类内容 |
| Check | 浴室仪式类收藏高,敏感肌类转化高,纯成分科普互动低 |
| Action | 保留仪式场景,强化敏感肌 RTB,减少纯知识科普,下一轮测试睡前场景 |

**关键洞察**: PDCA 让植愈坊不是凭感觉判断内容好坏,而是用数据和用户反馈找到下一轮传播重点。

### 示例 2 · SmallRig 新品发布 PDCA

> **案例可追溯报告**: ⚠️ **待人工校对** — 本示例的 SmallRig 内容需逐字与 `assets/_raw/cases/标杆案例/smallrig/*.md` 真实页对照。如案例 PDF 中无对应内容, 必须按 P0-6 4 层 Fallback 改用 Tier 2 经典案例或 Tier 3 抽象示例, 禁止 LLM 编造 SmallRig 案例。

背景: SmallRig 为某新机型相机笼做全球发布,目标是验证快速适配和创作者工作流叙事。

| 阶段 | 输出 |
|---|---|
| Plan | 发布首周完成新品页、KOL 测评、工作流短视频和社群反馈收集 |
| Do | 同步官网、YouTube、社群和电商详情页,统一 "快速进入工作状态" 信息 |
| Check | KOL 测评带来高信任,教程视频带来高停留,详情页参数段转化弱 |
| Action | 将参数改为场景化模块组合,把高频问题进入 FAQ 和下一版产品迭代 |

**关键洞察**: SmallRig 的 PDCA 不只优化传播,还把创作者反馈回流到产品和工作流表达。

## 常见误用

1. **只有 Plan 和 Do**: 计划完就执行,执行完就下一个项目 → 没有组织学习
2. **Check 只看曝光**: 不看转化、体验、情绪和用户反馈 → 误判传播效果
3. **Action 不标准化**: 复盘会上说得很好,没有进入 SOP → 下次继续重复错误
4. **指标不对应目标**: 目标是建立信任,却只看播放量 → 优化方向跑偏
5. **复盘变甩锅**: 只追责个人,不找流程和假设问题 → 团队不愿记录真实偏差
6. **周期过长**: 等年度结束才复盘 → 错过快速调整窗口
7. **不进入下一轮 Plan**: Action 清单没有负责人和截止时间 → 循环断掉
