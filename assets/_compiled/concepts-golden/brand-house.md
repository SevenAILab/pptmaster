# Brand House · 品牌屋

## 定义

品牌屋 (全称: **品牌信息屋 Brand Message House**) 是一套**战略信息的传达系统**。

**核心定位**: 品牌屋 = **战略意图 + 战术落地** = **对内组织统领 + 对外信息统一**。

- **对内**: 作为"企业宪法"保证组织效率, 做到"有法可依, 有法必依" — 提升组织效率即提升经营效率
- **对外**: 消费者跟品牌的每一个触点都直接影响认知, 统一的信息直接影响沟通效率

**结构**: 五层金字塔 — 上一层是下一层的**来源**, 下一层是上一层的**落地**。

```
              ┌──────────────────┐
              │  落地: 产品矩阵   │   ← 第 5 层
              ├──────────────────┤
              │  论据: RTB 体系   │   ← 第 4 层
              ├──────────────────┤
              │  论点: 核心价值   │   ← 第 3 层
              ├──────────────────┤
              │ 心智:用户+口号+调性│   ← 第 2 层
              ├──────────────────┤
              │  战略: 品牌定位   │   ← 第 1 层 (地基)
              └──────────────────┘
```

## 适用场景

- **品牌建设案的总骨架**: Sub-Agent ⑤ brand_building 必检字段"1 张品牌屋图"的产出依据
- **跨部门战略统一工具**: 让产品/营销/销售/客服共用一套"信息地图", 减少内部摩擦
- **新品牌从 0 到 1 体系化**: 把零散的定位/口号/产品决策收纳到一张图
- **现有品牌资产盘点 + 升级**: 检查 5 层是否有断裂/失焦, 找升级抓手
- **不适合**: 替代品牌定位案/产品屋/VI 手册 — Brand-House 是"目录式总图", 各层需要专项工具继续展开

## 使用步骤

### 第 1 步 · 第 1 层「战略 (品牌定位)」 — 五子问题回答

品牌定位是品牌屋的核心, 需要用**一句话**说明能解决什么问题。回答五个子问题:

| 子问题 | 解答方向 | 关联工具 |
|---|---|---|
| **对谁而言** | 目标消费者 | STP-T / Persona-5W2H |
| **你属于什么品类** | 消费者心智中的产品类别 | 品类战略 (艾·里斯) |
| **解决什么问题** | 消费者未被满足的核心需求 | JTBD / Pain-Gain-Map |
| **可以达到什么效果** | 消费者可感知的具体利益 | Value-Prop-Canvas |
| **因为你有什么支撑** | RTB 体系 (Reason to Believe) | RTB / SWOT-S |

**关键洞察**: 品类思维的本质是用户思维。**"类"是需求类别**, **"品"是产品如何满足这个需求**。消费者关心的是 **Why (消费者为何选你)**, 而非企业的 **What (企业在做什么)** 和 **How (企业是如何做的)**。

### 第 2 步 · 第 2 层「心智 (用户+口号+调性)」 — 把定位翻译给消费者

品牌定位是内部战略共识, 心智占据是把定位翻译成消费者**可感知、可记忆、好传达**的表达。三大要素:

**2.1 目标人群**
- 产品为需求服务, 需求来自特定人群
- 任何脱离目标人群谈价值的, 都是耍流氓
- **铁律**: 人群定错, 战略全错

**2.2 品牌口号**
- 把核心价值用**更高效**的方式跟消费者沟通
- 是直接跟消费者高频沟通的那句话
- 详见 `concepts-golden/slogan-7-principles.md`

**2.3 品牌调性**
- 如果说品牌口号决定了**说什么**, 品牌调性决定**用什么方式表达**
- 品牌调性 = 品牌的"人格特质", 决定消费者如何看待这个品牌
- 详见 `concepts-golden/aaker-brand-personality.md`

### 第 3 步 · 第 3 层「论点 (核心价值)」

把"我能给消费者什么"提炼为 3-5 个核心价值点 (Value Pillars), 每个价值点是 1 句话, 与第 1 层"解决什么问题/达到什么效果"对齐。

**示例**:
- 安全 (天然成分, 0 添加)
- 治愈 (情绪疗愈, 仪式感)
- 美感 (设计独特, 包装精致)

### 第 4 步 · 第 4 层「论据 (RTB 体系)」

每个核心价值点必须有对应的 RTB (Reason to Believe) — 让消费者**真信**的证据。

**RTB 来源**:
- **资质背书**: 创始人专业背景 / 行业认证 / 专利技术
- **第三方背书**: KOL/媒体/客户证言/获奖
- **数据证明**: 销量/复购率/用户增长
- **事实可验**: 配方公开 / 工艺透明 / 检测报告

每个论点都要配 1-3 个 RTB, 否则就是空话。

### 第 5 步 · 第 5 层「落地 (产品矩阵)」

把品牌承诺落到具体产品 SKU 矩阵 — 这一步是与产品屋的衔接点。

**产品矩阵设计原则**:
- 主打 SKU (引流款 / 爆款) → 体现核心价值
- 利润 SKU (复购款 / 高溢价) → 撑商业模式
- 形象 SKU (高端旗舰 / 限量) → 拉高调性
- 教育 SKU (低门槛体验) → 拓新

详见 `concepts-golden/product-house.md`

### 第 6 步 · 五层一致性审视 — 上下贯通

填完 5 层后, 必须做闭环审视:

| 检验问题 | 涉及层 |
|---|---|
| 心智表达 (口号/调性) 是否准确翻译战略定位? | 1 ↔ 2 |
| 核心价值是否真的回答了"解决什么问题"? | 1 ↔ 3 |
| RTB 是否真的能让消费者信核心价值? | 3 ↔ 4 |
| 产品矩阵是否真的承载了核心价值? | 3 ↔ 5 |
| 产品矩阵是否覆盖目标人群的全场景? | 2 ↔ 5 |

### 第 7 步 · 输出 — 1 张可视化品牌屋图

Sub-Agent ⑤ 渲染时强烈推荐版式 **S05 Three Layers** (上层战略 / 中层心智+论点 / 下层论据+落地) 或 **S17 System Diagram** (5 层金字塔结构)。

## 输入输出示例

### 示例 1 · 植愈坊 (247 书贯穿教学案例, Claude 基于 247 各章推演的品牌屋)

> **案例可追溯报告**: 植愈坊是 Seven 247 书《AI 实战, 从 0 到 1 打造你的品牌》全书贯穿的**教学性虚拟案例**, 247 书在 ch04/05/06/07/08/09/10 各章对其做了系统化推演 (用户分析/SWOT/STP/BMC/价值主张/口号/视觉)。本示例的品牌屋是 Claude 根据 247 书各章推演结果拼合而成 — 不是"真实成功品牌的品牌屋", 而是"按方法论应该长这样"的范式示例。所有底层素材 (人群/口号/产品矩阵) 都能在 247 书相应章节追溯。

```
第 5 层 · 落地 (产品矩阵):
  - 主打: 「都市疗愈系列」精油手工皂 4 款 (引流爆款)
  - 利润: 季节限定「24 节气疗愈皂」(高溢价复购)
  - 形象: 与艺术家联名礼盒 (拉调性)
  - 教育: 单块体验装 + 9.9 元试用

第 4 层 · 论据 (RTB):
  - "天然成分": 6 国有机精油协会认证, 配方公开, 每批 SGS 检测
  - "情绪疗愈": 创始人芳疗师 5 年临床 + 100+ 用户疗愈日记证言
  - "设计美感": 创始人设计师背景 + 包豪斯极简获奖包装
  
第 3 层 · 论点 (核心价值):
  - 价值 1: 安全 (天然 + 0 化学添加)
  - 价值 2: 治愈 (情绪疗愈 + 仪式感)
  - 价值 3: 美感 (独特设计 + 仪式包装)

第 2 层 · 心智:
  - 目标人群: 都市年轻女性, 25-35 岁, 注重情绪疗愈与生活仪式感
  - 品牌口号: "洗一次, 疗愈一次"
  - 品牌调性: 温柔 + 文艺 + 专业 (Aaker: 真诚 + 精致)

第 1 层 · 战略 (品牌定位):
  对都市年轻女性 (target),
  我们是手工皂品类中 (category),
  唯一把"情绪疗愈 + 设计仪式感"做透的品牌 (key benefit),
  因为创始人芳疗师专业 + 设计师视觉双背书 + 严格原料把控 (RTB)。
```

### 示例 2 · SmallRig MI 升级真实品牌屋 (100% 还原 page-124)

> **案例可追溯报告**: 本示例所有内容**完全来自** `assets/_raw/cases/标杆案例/smallrig/page-124.md` 的 OCR 原文 (该页是 SmallRig 内部对外发布的"SmallRig 品牌屋"全图), 未做任何 LLM 改写或扩展。

**SmallRig 实际呈现的是 9 层信息屋** (比通用 5 层金字塔更细, 但本质可映射到 5 层):

| SmallRig 实际层级 | 内容 | 对应通用 5 层 |
|---|---|---|
| **品牌定位** (顶部三角) | 「全球影像场景产品生态开创者」 | 第 1 层 战略 |
| **使命愿景** | 使命: 突破影像边界, 让拍摄更自由 / 愿景: 每个人都能实现创作梦想 | 第 1 层 战略 (灵魂层) |
| **主张/口号** | FREE YOUR DREAM 自由创想 | 第 2 层 心智 (口号) |
| **产品口号** | Rig UP | 第 2 层 心智 (口号) |
| **命名释义** | RIG = R: Redefinition 重新定义 + I: Imagination 想象力 + G: Gear 装备 | 第 2 层 心智 (品牌叙事) |
| **品牌人群** | 影像内容创作者 | 第 2 层 心智 (用户) |
| **功能利益 / 情感利益** | 功能: 实现更多创作可能和更高创作能力 / 情感: 对释放灵感和创作自由的追求 | 第 3 层 论点 |
| **差异 RTB** | 全生态 / 全场景 / 全兼容 / 快制造 | 第 4 层 论据 |
| **产品系列** | 相机支撑与稳定 / 储能解决方案 / 手机支撑与稳定 / 灯光与控制系统 | 第 5 层 落地 |

**视觉规范** (来自 page-124 注释): 橙色为对外传播 / 灰色为对内共识 — 这是 SmallRig 品牌屋的"对内对外双轨"实操体现。

**SmallRig 品牌屋的设计精妙之处** (Sub-Agent 学习要点):

1. **第 1 层 战略锐度**: "全球影像**场景**产品**生态**开创者" — 锁定"场景"和"生态"两个差异词, 区别于"配件供应商"和"产品商"
2. **第 2 层 心智双口号**: 品牌主张 (FREE YOUR DREAM, 给消费者) + 产品口号 (Rig UP, 给行业/创作者) — 分场景使用
3. **第 2 层 命名翻译**: RIG 这三个字母逆向赋予品牌叙事 (Redefinition + Imagination + Gear), 把"工具品牌名"升级为"创作哲学"
4. **第 3 层 双利益并陈**: 功能利益 (创作能力) + 情感利益 (创作自由) — 同时打"理性 + 感性"两根弦
5. **第 4 层 RTB 4 个字精炼**: 全生态 / 全场景 / 全兼容 / 快制造 — 都是"全+快"句式, 易记易传播
6. **第 5 层 产品 4 大系列**: 不按 SKU 列, 按"使用场景"切, 与第 1 层"场景生态"呼应

## 常见误用

1. **5 层断裂**: 下一层不是上一层的真实落地 (e.g. 心智说"年轻女性", 产品矩阵全是中年男人配置) → 品牌屋名存实亡
2. **战略层不够锐**: 第 1 层定位陈述模糊/笼统 (e.g. "做最好的品牌") → 下面 4 层都跟着虚化
3. **论点 = 空话**: 第 3 层写"高品质/创新/用户至上"等口水词 → 没有具体价值点
4. **论据缺失**: 第 4 层 RTB 没写 → 消费者凭什么信你?
5. **产品矩阵 = SKU 罗列**: 第 5 层简单列产品名, 没标注每个 SKU 对应承载的核心价值 → 产品策略没和品牌策略对齐
6. **只对内不对外**: 品牌屋只在战略文档里, 没翻译成销售/客服/营销可用的话术 → "企业宪法"成了一张挂图
7. **一次写完归档**: 不随业务/市场动态更新 → 错过升级机会 (SmallRig MI 升级正是品牌屋的重大迭代)
8. **混淆产品屋和品牌屋**: 产品屋讲"产品如何被定义", 品牌屋讲"品牌如何被认知" — 详见 `concepts-golden/product-house.md`
9. **跨部门未共识**: 品牌屋只在市场部, 产品/销售/客服各执一词 → 对外信息不统一, 内耗加大
