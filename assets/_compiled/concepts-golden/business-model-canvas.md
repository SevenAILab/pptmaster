# Business Model Canvas · 商业模式画布

## 定义

商业模式画布 (BMC) 是由 **Alexander Osterwalder** 与 Yves Pigneur 在《商业模式新生代》(2010) 提出的可视化战略工具,把复杂商业模式分解为 **9 个相互关联的构造块**,帮助系统性思考"企业如何**创造价值、传递价值、获取价值**"。

**第一性原理**: 商业模式的本质是 **买卖双方基于价值的交换**。极简定义 = "一个企业如何发现价值、创造价值和获取价值的过程"。大白话: 你发现什么需求 → 用什么产品满足它 → 怎么从中赚到钱。

**铁律**: 好商业模式 = 说的人好表达 + 听的人好理解。大道至简, 复杂的商业模式都不是好的商业模式。

## 适用场景

- **品牌定位案的可行性检验**: Sub-Agent ④ 必检字段"1 张商业模式画布"的产出依据
- **从品牌理想到商业现实的桥梁**: 让"想去哪里" (定位) 落到"如何活着到那里" (生意)
- **早期商业模式设计**: 0-1 创业期一次性铺出 9 模块的雏形
- **现有商业模式审视**: 找断点 (某个模块虚弱) / 找跃迁机会 (某个模块升级)
- **不适合**: 替代财务建模/单元经济模型 (LTV/CAC) — BMC 是定性框架, 不是定量预测

## 使用步骤

### 第 1 步 · 填写 9 大构造块 (按顺序)

按"先客户后供给, 先收入后成本"的逻辑展开:

| 序 | 模块 | 缩写 | 核心问题 | 与品牌定位的关系 |
|---|---|---|---|---|
| 1 | **客户细分** | CS | 我们为谁创造价值? 最重要的客户是谁? | 对应/细化品牌定位的目标客户 |
| 2 | **价值主张** | VP | 我们为客户传递什么价值? 解决什么问题/满足什么需求? | BMC 的核心, 来自上一章价值主张, 详见 Value-Prop-Canvas |
| 3 | **渠道通路** | CH | 我们如何触达客户并传递价值? | 对应品牌策略的渠道选择 |
| 4 | **客户关系** | CR | 与每个客户细分群体建立何种关系? | 个性化/自助/社群/共创 — 体现品牌个性 |
| 5 | **收入来源** | R$ | 从每个客户群体获取哪些收入? 定价机制? | 产品销售/订阅/服务费/许可/广告 |
| 6 | **核心资源** | KR | 实现 VP 需要哪些核心资产? | 物理/知识产权/人力/财务 — 与 SWOT 优势 S 强关联 |
| 7 | **关键业务** | KA | 实现 VP 需要执行哪些关键活动? | 生产/研发/营销/平台/供应链 |
| 8 | **重要伙伴** | KP | 需要哪些合作伙伴和供应商? | 优化模式/降低风险/获取资源 |
| 9 | **成本结构** | C$ | 运营该模式需要承担哪些主要成本? | 固定/可变/规模/范围经济 — 与 SWOT 劣势 W 强关联 |

### 第 2 步 · 检验"流动效率" (基于摘要 08 的实战派视角)

BMC 不是 9 个模块的孤立堆砌, 而是要组合成一条完整商业链:

```
需求 → 产品 → 流量与渠道 → 转化 → 复购
```

五大环节间的**流动效率越高, 商业模式就越好**。每个环节问:
- **需求**: 是真需求还是伪需求? 大需求还是小需求? 短期还是长期? **无需求, 不商业**
- **产品**: 你拿什么来满足这个需求? 产品是否真的能解决核心痛点?
- **流量与渠道**: 用户从哪来? 渠道是否能有效触达目标客户?
- **转化**: 流量到付费的转化率? 客户决策路径是否顺畅?
- **复购**: 客户为什么会再来? 这是商业可持续的关键

### 第 3 步 · 检验模块间的逻辑一致性 — 见 247 书 §7.3

填完 9 模块后必须做一致性审视,绕一遍闭环:

| 检验问题 | 涉及模块 |
|---|---|
| 价值主张真的传递了吗? | VP → CS / CH / CR |
| 客户关系策略与客户细分和价值主张匹配吗? | CS / VP / CR |
| 渠道通路能有效传递价值主张吗? | VP / CH |
| 收入来源能覆盖成本结构吗? | R$ / C$ |
| 核心资源能持续支撑关键业务和价值主张吗? | KR / KA / VP |
| 重要伙伴是否真的优化模式/降低风险/获取资源? | KP / KR / C$ |

### 第 4 步 · 识别关键假设与最高风险环节

填完后问:
- **哪些模块基于假设?** (e.g. 假设用户愿意为 "疗愈感" 支付 30% 溢价? 假设小红书引流转化率达 X%?)
- **这些假设需要哪些 MVP 验证?**
- **哪个模块风险最高?** (失败会导致商业模式整体崩塌)

### 第 5 步 · 输出 — 1 张可视化 BMC + 关键假设清单 + 验证 plan

Sub-Agent ④ 渲染时强烈推荐版式 **S17 System Diagram** (3-3-3 九宫格布局) 或 **S05 Three Layers** (上层 CS/VP 价值层 + 中层 CH/CR/R$ 客户层 + 下层 KR/KA/KP/C$ 后台层)。

## 输入输出示例

### 示例 1 · Nespresso "系统型"商业模式 (来自 247 书 §7.2 案例)

> **案例可追溯报告**: 真实公知品牌。本示例 9 模块内容 (VP/CS/KR/R$/CH/KP/CR/KA/C$) **完全来自** `assets/_raw/books/0to1-brand/ch07-bmc.md` L170-186 原文 (Seven 在 §7.2 给出的 Nespresso "系统"商业模式案例)。"剃刀-刀片"模式 (Razor and Blades) 类比是 Claude 基于该原文做的商业模式范式归类, 不是 247 书原文用词。

```
VP 价值主张:    "在家轻松享用高品质意式浓缩咖啡"
CS 客户细分:    追求品质和便利的家庭及办公室用户
KR 核心资源:    专利咖啡胶囊 + 机器技术 + 高端品牌形象
R$ 收入来源:    ⚠️ 关键 — 咖啡机销售 (一次性) + 持续复购的咖啡胶囊 (高利润, 锁定用户)
CH 渠道通路:    精品店 + 官网 + 俱乐部 — 营造高端体验
KP 重要伙伴:    咖啡农场 + 机器制造商 (OEM)
CR 客户关系:    俱乐部会员制 (订阅 + 个性化推荐)
KA 关键业务:    胶囊研发 + 全球分销 + 用户体验设计
C$ 成本结构:    胶囊生产边际成本低, 主要成本在品牌建设和门店运营
```

**模式关键**: "机器 + 胶囊"的系统性锁定 → 一次性高客单 (机器) + 高频高粘性 (胶囊) → 构建强大用户粘性和盈利能力。这是经典的"剃刀-刀片"模式 (Razor and Blades) 在咖啡品类的应用。

### 示例 2 · 植愈坊推演 (Claude 基于 247 书 ch07 §7.2 prompt 推演)

> **案例可追溯报告**: 植愈坊是 Seven 247 书全书贯穿的**教学性虚拟案例**。本示例 9 模块内容是 Claude 基于以下两份原文综合推演而成: ① `ch07-bmc.md` §7.2 L72-149 (Seven 给出的"AI 辅助 BMC 推演植愈坊"的 Prompt 模板) ② 247 书 ch04/05/06 植愈坊的用户/SWOT/STP 推演结论 (作为 9 模块的输入数据)。247 书 §7.2 **没有直接给出植愈坊完整 9 模块填充结果** (只给了引导 Prompt), 9 模块的具体内容由 Claude 基于全书植愈坊一致性推演。3 个关键假设 (30% 溢价 / 3% 转化 / 35% 复购) 是 Claude 综合判断, 不是 247 原文数据。**禁止包装成"真实数据"**。

```
CS:  注重情绪疗愈和生活仪式感的都市年轻女性 (核心) + 礼品场景买家 (次要)
VP:  "洗一次, 疗愈一次" — 天然芳疗手工皂 + 治愈体验仪式感
CH:  小红书种草 + 微信私域复购 + 淘宝旗舰店 (后期)
CR:  KOC 共创 + 私域社群 (情绪疗愈话题深度参与)
R$:  产品销售 (主) + DIY 材料包 (拓展) + 付费芳疗咨询 (溢价)
KR:  创始人芳疗师专业知识 + 设计能力 + 独家精油配方 + 品牌声誉 (待建)
KA:  产品研发 + 小红书内容创作 + 社群运营 + 供应链管理
KP:  有机原料供应商 + 小红书 KOC + 物流伙伴 + 环保包装供应商
C$:  原材料 (高) + 内容营销 (中) + 人工 (低) + 平台佣金 (中)
```

**关键假设**: ① 都市年轻女性愿为"疗愈感"付 30% 溢价 ② 小红书 KOC 种草转化率 ≥ 3% ③ 私域复购率 ≥ 35%。这 3 个假设构成商业模式的支柱, 必须 MVP 验证。

### 示例 3 · 苹果 MacBook Pro Touch Bar (反例, 来自摘要 08)

> **案例可追溯报告**: 真实公知品牌反例。本示例核心叙述 ("F1-F12 改 Touch Bar / 重度用户讨厌打断感 / 最终被苹果取消") **完全来自** `assets/_raw/methodologies/summaries/08-business-model.md` L40-42 原文 (摘要作者作为亲历用户的反思)。VP/CS/KA 的 BMC 三模块映射是 Claude 基于该原文做的 BMC 框架化归因, 不是摘要原文的 BMC 拆解。

苹果当时把 F1-F12 按键改为 Touch Bar 触控屏, 自认为"剪视频显示时间轴, 打字显示表情"很酷。

**BMC 检验**:
- VP 失效: 价值主张是"提升专业用户效率", 但 Touch Bar 实际是**降低效率** (盲按消失, 必须低头看)
- CS 不匹配: 重度键盘用户最讨厌打断感, 这是错配最深的客户群
- KA 浪费: 大量研发投入了一个用户不要的功能

→ 最终被苹果取消, 验证了"需求是商业模式的前提, **无需求不商业**"。提醒 **BMC 第一步 CS-VP 必须建立在真实需求上, 不能是产品经理自嗨**。

## 常见误用

1. **9 模块填了就完事**: 不做"流动效率"检验, 不做"逻辑一致性"审视 → BMC 退化为"漂亮的九宫格"
2. **VP 写得空泛**: e.g. "提供最好的服务" → 没有具体价值点 → 后面 8 模块都跟着虚化
3. **R$ 和 C$ 严重失衡**: 收入来源不清, 或成本驱动因素没识别 → 商业不可持续
4. **KR 与 KA 错配**: 关键活动需要的资源 (KR) 实际没有 → 模式跑不起来
5. **CR 与 VP 矛盾**: 价值主张说"个性化深度服务", 客户关系却是"自助低成本" → 客户体验崩塌
6. **缺少假设识别和验证**: 9 模块都填了, 但没标注"哪些是假设", 看不出最高风险点 → 没法做 MVP 验证
7. **静态化使用**: 一次画完归档, 不随市场动态更新 → 错过升级机会 (Nespresso 也在持续迭代)
8. **复杂化倾向**: 加越来越多模块和细节 → 违反"大道至简"原则 → 团队无法对齐
9. **从产品出发**: 不是"用户需求 → 产品 → 商业模式", 而是"我有这个产品 → 怎么硬塞商业模式" → Touch Bar 式失败
