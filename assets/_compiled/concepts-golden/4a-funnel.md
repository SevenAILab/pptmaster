# 4A Funnel · 4A 行为漏斗

## 定义

4A-Funnel 是 PPTAgent 内部用于品牌策略诊断的行为漏斗: 把用户从"知道品牌"到"愿意行动并推荐"分为 4 个阶段。

```text
Awareness 认知 → Attitude 态度/好感 → Action 行动/购买 → Advocacy 拥护/推荐
```

本地材料中直接出现的是"销售漏斗 / 品牌销售漏斗 / 消费者购买漏斗",并将问题拆成知晓、考虑、评估、购买、私域、忠诚等层级。4A-Funnel 在这里承担归一化作用: 用更短的 4 层结构把年度任务、传播触点和用户旅程连起来。

## 适用场景

- **年度品牌任务拆解**: Sub-Agent ⑥ 用漏斗定位增长问题,如认知不足、信任不足、转化不足、复购不足
- **消费者旅程压缩诊断**: 把复杂旅程归纳成 4 个可量化阶段
- **IMC 传播分工**: 不同内容/渠道对应不同漏斗任务
- **品牌建设检查**: 看品牌资产是否既能提升心智,也能推动行动
- **不适合**: 替代完整用户旅程；漏斗是诊断框架,不是端到端体验地图

## 使用步骤

### 第 1 步 · 定义 4A 阶段口径

先统一团队的阶段定义:

| 阶段 | 核心问题 | 常见指标 |
|---|---|---|
| Awareness 认知 | 用户知道我吗? 知道我代表什么吗? | 曝光、搜索指数、品牌提及、认知度 |
| Attitude 态度 | 用户对我有好感和信任吗? | 收藏、互动、咨询、好评、内容完读 |
| Action 行动 | 用户是否试用/购买/留资? | 点击、加购、转化率、首购成本 |
| Advocacy 拥护 | 用户是否复购和推荐? | 复购率、NPS、UGC、私域活跃、推荐 |

### 第 2 步 · 把用户旅程映射到漏斗

把认知、了解、评估、购买、使用、复购、推荐等旅程阶段合并到 4A。目的不是丢细节,而是快速判断主要断点在哪里。

输出标准: 用户旅程阶段 → 4A 阶段 → 当前证据 → 主要问题。

### 第 3 步 · 识别漏斗断点

年度规划资料强调"销售倒逼品牌": 用漏斗梳理问题和任务。常见断点:
- Aware 弱: 用户不知道,或知道但心智模糊
- Attitude 弱: 有曝光但不信任、不喜欢、不理解差异
- Action 弱: 有好感但不下单,价格/风险/渠道/转化页有阻碍
- Advocacy 弱: 有首购但不复购、不推荐,体验或关系经营不足

### 第 4 步 · 为每层配置品牌任务

每层只配置 1-3 个关键任务:

| 阶段 | 任务例子 |
|---|---|
| Awareness | 品牌出圈、明确品类、提高第一联想 |
| Attitude | 强化信任状、讲清价值主张、建立美誉度 |
| Action | 降低试用门槛、优化购买路径、推出首购理由 |
| Advocacy | 私域运营、会员权益、用户故事、推荐机制 |

### 第 5 步 · 分配内容、渠道与指标

将每层任务落到 IMC:
- Awareness: PR、达人扩散、短视频、事件话题
- Attitude: 深度内容、测评、创始人故事、RTB 证明
- Action: 直播、电商详情页、试用装、限时权益
- Advocacy: 社群、复购计划、UGC 征集、老客推荐

### 第 6 步 · 输出漏斗诊断页

推荐输出: 4 层漏斗 + 每层问题 + 任务 + 指标 + 关键动作。Sub-Agent 渲染可用 **S15 Matrix** 或 **S17 System Diagram**。

## 输入输出示例

### 示例 1 · 年度品牌任务漏斗

> **案例可追溯报告**: ⚠️ **待人工分类** — 请按 P0-6 4 层 Fallback 决策树标注本案例性质 (Tier 1 真实案例 / Tier 2 行业经典 / Tier 3 抽象示例 / Tier 4 教学案例), 并补全 source 路径 + 行号。

来自年度规划资料的核心逻辑: 销量倒逼品牌,增长点 = 拉新率 + 转化率 = 认知度 + 美誉度。

| 4A | 面临问题 | 品牌任务 | 指标 |
|---|---|---|---|
| Awareness | 基础薄弱,心智模糊 | 品牌出圈,明确品类词 | 搜索量、自然提及、曝光 |
| Attitude | 感知缺失,信任不足 | 强化 RTB,建立美誉度 | 收藏率、咨询率、好评 |
| Action | 增长乏力,转化不稳 | 降低首购门槛,优化购买链路 | 首购转化率、CAC |
| Advocacy | 基盘收缩,复购不足 | 私域经营,用户故事,会员机制 | 复购率、NPS、UGC |

**关键洞察**: 漏斗不是销售部门的表,而是品牌任务的拆解器。品牌工作必须解释它解决哪一层断点。

### 示例 2 · 植愈坊冷启动 4A 漏斗 (247 书贯穿教学案例)

> **案例可追溯报告**: 植愈坊是 Seven 247 书全书贯穿的**教学性虚拟案例** (非真实商业体)。具体出处行号待 Codex 后续按 P0-4 模板填充, 当前仅标注教学属性。**禁止包装成"真实品牌成功案例"**。

| 4A | 当前断点 | 动作 |
|---|---|---|
| Awareness | 目标用户不知道"情绪疗愈洁肤"这个新表达 | 小红书内容用"下班后 15 分钟疗愈洗澡"切入 |
| Attitude | 用户怀疑天然/精油/敏感肌是否可信 | 芳疗师资质 + 成分公开 + 敏感肌测试 |
| Action | 单块价格较高,首次尝试风险大 | 3 款小样组合 + 肤质场景推荐 |
| Advocacy | 首购后缺少分享理由 | 礼盒故事卡 + 用户疗愈日记征集 |

**关键洞察**: 植愈坊首要不是马上砸转化,而是先解决 Attitude 层的可信度,否则 Action 层会被价格阻挡。

## 常见误用

1. **把漏斗当线性真理**: 用户不一定按顺序前进 → 漏斗用于诊断,不是现实的唯一路径
2. **只看 Action**: 只盯购买转化,忽视认知和态度 → 短期促销越做越贵
3. **每层指标混乱**: 用曝光评估转化,用销售评估心智 → 判断失真
4. **任务不分层**: 所有渠道都喊同一句促销话术 → 认知、信任、转化无法协同
5. **忽略 Advocacy**: 首购后没有复购和推荐设计 → 品牌无法沉淀资产
6. **把 AIDA/AIDMA/4A 混用不解释**: 团队口径不一 → 方案评审时各说各话
7. **没有用户证据**: 漏斗断点靠猜 → 应结合旅程、搜索、咨询、转化数据验证
