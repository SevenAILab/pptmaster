# User Journey · 用户旅程地图

## 定义

用户旅程 (Customer Journey) 是用户从首次意识到需求或接触品牌,到了解、比较、购买、使用、复购、推荐或流失的全过程路径。用户旅程地图 (CJM) 则把每个阶段的行为、想法、情绪、触点、痛点和机会点可视化。

247 书给出的核心价值是: 让团队"穿上用户的鞋子走路",跳出"我觉得"的品牌视角,完整审视用户与品牌互动的每一个环节。它不是简单的流程图,而是体验诊断工具。

## 适用场景

- **消费者洞察必检**: Sub-Agent ① 用它输出触点、摩擦点和机会点
- **转化问题诊断**: 看清用户卡在认知、考虑、购买、使用还是复购阶段
- **IMC 传播规划**: 把不同阶段的内容、渠道和信息任务分配清楚
- **品牌体验一致性检查**: 检查广告、详情页、包装、客服、社群是否说同一件事
- **不适合**: 替代真实访谈和行为数据；AI 可以搭框架,但不能完全替代用户共创

## 使用步骤

### 第 1 步 · 选定 Persona 与核心场景

旅程必须绑定一个具体画像和一个具体任务。不要写"所有用户购买全过程",而要写"林悦溪第一次在小红书看到植愈坊后完成试用购买的旅程"。

输出标准: Persona + JTBD + 起点/终点。

### 第 2 步 · 划分关键阶段

可按行业调整,但电商品牌常用阶段为:

| 阶段 | 用户问题 |
|---|---|
| 认知/发现 | 我为什么会注意到这个问题或品牌? |
| 了解/研究 | 它是什么,是否适合我? |
| 考虑/评估 | 它和替代方案相比有什么不同? |
| 购买/决策 | 我为什么现在下单? 风险是什么? |
| 使用/体验 | 它是否兑现了承诺? |
| 售后/忠诚 | 我会不会复购? 为什么留下? |
| 推荐/分享 | 我愿不愿意告诉别人? 用什么话说? |

### 第 3 步 · 填行为、想法、情绪、触点

每个阶段至少填 4 类信息:
- **行为**: 用户具体做了什么
- **想法**: 用户脑子里在问什么
- **情绪**: 兴奋、怀疑、焦虑、失望、安心等
- **触点**: 小红书、搜索、朋友推荐、详情页、客服、包装、社群等

### 第 4 步 · 标记痛点与摩擦点

247 书强调用户旅程的价值是发现机会点和摩擦点。摩擦点要写具体,例如"详情页没有敏感肌测试依据"比"信任不足"更可行动。

输出标准: 每阶段 1-3 个摩擦点 + 影响程度 + 可验证证据。

### 第 5 步 · 把机会点转成动作

每个机会点必须能落到产品、内容、渠道或服务动作:
- 内容动作: 写什么选题、回应什么疑虑
- 渠道动作: 哪个平台承担哪个阶段任务
- 产品动作: 哪个功能/包装/说明改进
- 服务动作: 哪个触点需要客服或社群介入

### 第 6 步 · 输出旅程地图

推荐输出为矩阵表: 阶段 × 行为/想法/情绪/触点/痛点/机会点/指标。Sub-Agent 渲染时可用 **S15 Matrix** 或 **S07 Timeline**。

## 输入输出示例

### 示例 1 · 植愈坊林悦溪首购旅程 (247 书贯穿教学案例)

> **案例可追溯报告**: 植愈坊是 Seven 247 书全书贯穿的**教学性虚拟案例** (非真实商业体)。具体出处行号待 Codex 后续按 P0-4 模板填充, 当前仅标注教学属性。**禁止包装成"真实品牌成功案例"**。

| 阶段 | 行为/触点 | 情绪 | 摩擦点 | 机会点 |
|---|---|---|---|---|
| 认知 | 小红书刷到"下班后的疗愈洗澡仪式" | 被击中 | 不确定是不是广告软文 | 用真实用户日记式内容开场 |
| 了解 | 搜成分、看创始人介绍 | 好奇但谨慎 | 天然/精油概念真假难辨 | 展示芳疗师资质和成分透明卡 |
| 评估 | 对比 Lush/普通沐浴露/手工皂店 | 犹豫 | 价格高,怕不适合敏感肌 | 试用装 + 敏感肌 FAQ |
| 购买 | 下单小样组合 | 期待 | 物流和气味不可预期 | 小样命名清晰,降低选择成本 |
| 使用 | 晚上洗澡体验香气和肤感 | 放松 | 起泡和留香若不达预期会失望 | 包装内放"15 分钟疗愈仪式"卡 |
| 复购 | 加入私域,看季节限定 | 安心 | SKU 太多可能选择困难 | 按情绪场景推荐复购套装 |
| 推荐 | 晒浴室照片/送朋友 | 自我表达 | 缺少好分享的话 | 礼盒与口号强化社交传播 |

**关键洞察**: 植愈坊的关键摩擦不是"用户不知道皂是什么",而是"天然、温和、疗愈是否可信"。RTB、试用装和内容选题要集中解决这个摩擦。

### 示例 2 · SmallRig 创作者购置相机笼旅程

> **案例可追溯报告**: ⚠️ **待人工校对** — 本示例的 SmallRig 内容需逐字与 `assets/_raw/cases/标杆案例/smallrig/*.md` 真实页对照。如案例 PDF 中无对应内容, 必须按 P0-6 4 层 Fallback 改用 Tier 2 经典案例或 Tier 3 抽象示例, 禁止 LLM 编造 SmallRig 案例。

| 阶段 | 行为/触点 | 情绪 | 摩擦点 | 机会点 |
|---|---|---|---|---|
| 认知 | 新相机发布后看 YouTube/B 站装机视频 | 兴奋 | 不知道哪些配件必买 | 发布机型适配清单 |
| 研究 | 看兼容性、材质、重量、安装方式 | 专业谨慎 | 型号复杂,担心不兼容 | 机型筛选器 + 工作流图 |
| 评估 | 对比 Tilta/原厂/无品牌配件 | 价格敏感 | 性能差异不直观 | 创作者真实项目案例 |
| 购买 | Amazon/独立站/经销商下单 | 希望稳定 | 到货时效影响项目 | 明确库存与发货时效 |
| 使用 | 拍摄现场快速安装 | 紧张 | 拆装慢会影响交付 | 快装结构与教程视频 |
| 忠诚 | 继续补灯光/麦克风/监视器配件 | 信任 | 跨产品线选择困难 | 以场景打包工具套装 |

**关键洞察**: SmallRig 的旅程核心是"项目交付安全感",因此触点内容要从参数介绍升级为工作流可靠性证明。

## 常见误用

1. **阶段太粗**: 只写"购买前/购买中/购买后" → 找不到具体摩擦点
2. **只写品牌动作**: 写我们投广告、我们发内容,不写用户想法和情绪 → 仍是品牌视角
3. **没有 Persona**: 不同人群旅程不同,硬画一条通用路径 → 触点策略失真
4. **把触点当渠道清单**: 罗列小红书/抖音/淘宝,不说明阶段任务 → 渠道无法分工
5. **机会点不落地**: 写"提升体验"这种空话 → 没有产品/内容/服务动作
6. **忽略负面路径**: 只画理想购买路径,不画流失、退货、投诉 → 关键风险消失
7. **一次画完不验证**: 不用真实数据修正 → 旅程图变成会议室想象
