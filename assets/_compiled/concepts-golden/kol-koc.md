# KOL/KOC 合作 · KOL-KOC

## 定义

KOL/KOC 合作是品牌借助关键意见领袖和关键意见消费者进行内容种草、信任背书、口碑扩散和销售转化的传播方法。

本概念来自 MAP-STRUCTURE 05-传播层的 "KOL/KOC 合作",属于全景图新增候选,缺少 Seven 247 书中的独立专章。但在本地材料中,KOL/KOC 多次出现在渠道选择、用户洞察、品牌策略和小红书 KFS 模型中: 新品牌可以通过内容种草、达人合作和用户反馈建立信任,也可以用 KOL/KOC 作为新品上市与冷启动的关键杠杆。

KOL 更偏影响力和专业/圈层背书,KOC 更偏真实体验和信任感。对新品牌来说,二者不是越多越好,而是要服务目标人群、平台场景和品牌定位。

## 适用场景

- **新品冷启动**: Sub-Agent ⑥ 通过 KOC 试用和 KOL 背书获得首批信任
- **传播年度规划**: 设计达人矩阵、内容主题、预算和节点
- **竞品传播分析**: Sub-Agent ③ 拆解竞品合作达人类型、内容角度和转化路径
- **品牌证据建设**: Sub-Agent ⑤ 将测评、体验和用户反馈沉淀为 RTB
- **不适合**: 定位未清、产品体验未验证时大规模投达人；会放大问题而不是解决问题

## 使用步骤

### 第 1 步 · 明确合作目标

判断本轮合作要解决认知、信任、种草、转化、口碑还是内容资产问题。

输出标准: 合作目标 + 核心指标。

### 第 2 步 · 定义达人分层

按平台、垂类、人设、粉丝规模、内容质量、互动质量、商业化程度划分 KOL、KOC 和专家型创作者。

输出标准: 达人分层模型。

### 第 3 步 · 设定筛选标准

检查人群匹配、品牌调性、内容可信度、历史合作、评论质量、风险记录和报价。

输出标准: 达人筛选评分表。

### 第 4 步 · 设计内容 Brief

给出品牌主张、产品卖点、RTB、使用场景、禁用话术、必须披露的信息和创作自由边界。

输出标准: KOL/KOC 内容 Brief。

### 第 5 步 · 组合投放和承接

设计头部造势、腰部解释、尾部铺量或 KOC 真实体验,同时安排搜索、电商、私域和客服承接。

输出标准: 达人矩阵 + 渠道承接表。

### 第 6 步 · 复盘内容和转化

复盘曝光、互动、收藏、搜索、进店、成交、评论反馈和可复用内容资产。

输出标准: 投放复盘表 + 优质内容资产库。

## 输入输出示例

### 示例 1 · 植愈坊 KOL/KOC 种草 (247 书贯穿教学案例)

> **案例可追溯报告**: 植愈坊是 Seven 247 书全书贯穿的**教学性虚拟案例** (非真实商业体)。具体出处行号待 Codex 后续按 P0-4 模板填充, 当前仅标注教学属性。**禁止包装成"真实品牌成功案例"**。

背景: 植愈坊要验证 "睡前疗愈洁肤" 场景,预算有限。

| 模块 | 输出 |
|---|---|
| 合作目标 | 建立敏感肌用户信任,获取真实试用反馈 |
| 达人分层 | 芳疗师/成分科普 KOL 2 位,生活方式 KOC 30 位 |
| 内容角度 | 睡前仪式、香气体验、敏感肌温和测试、手工皂使用误区 |
| Brief 要点 | 不夸大疗效,必须说明适用肤质和试用感受 |
| 承接 | 小红书搜索词、淘宝详情页、私域咨询 FAQ |
| 指标 | 收藏率、评论问题、试用申请、复购意愿 |

**关键洞察**: 植愈坊更适合专家信任 + KOC 真实体验,而不是一上来追求头部达人声量。

### 示例 2 · SmallRig 创作者合作

> **案例可追溯报告**: ⚠️ **待人工校对** — 本示例的 SmallRig 内容需逐字与 `assets/_raw/cases/标杆案例/smallrig/*.md` 真实页对照。如案例 PDF 中无对应内容, 必须按 P0-6 4 层 Fallback 改用 Tier 2 经典案例或 Tier 3 抽象示例, 禁止 LLM 编造 SmallRig 案例。

背景: SmallRig 发布新机型适配套件,需要让创作者相信产品可靠、安装高效。

| 模块 | 输出 |
|---|---|
| 合作目标 | 抢占新机适配心智,证明工作流完整性 |
| 达人分层 | 专业摄影 YouTuber、器材测评 KOL、垂直社群 KOC |
| 内容角度 | 新机开箱适配、安装流程、实拍稳定性、套件组合推荐 |
| Brief 要点 | 展示真实安装和场景使用,避免只拍产品静物 |
| 承接 | 官网机型页、Amazon 套装页、安装教程和社群答疑 |
| 指标 | 视频完播、搜索增长、套装转化、社群问题减少 |

**关键洞察**: SmallRig 的达人合作不是 "找人带货",而是让专业创作者替用户验证工作流。

## 常见误用

1. **只按粉丝量选人**: 粉丝大但人群不匹配 → 声量有了,转化很弱
2. **Brief 写成硬广脚本**: 达人失去真实感 → 用户不信
3. **没有承接路径**: 内容种草后搜不到、买不到、问不到 → 流量浪费
4. **KOL 和 KOC 角色混乱**: 让 KOC 承担权威背书,让 KOL 做低价晒单 → 效率下降
5. **夸大产品效果**: 短期转化可能上升,长期信任受损 → 品牌资产被透支
6. **只看曝光不看评论**: 评论里的顾虑和真实问题被忽略 → 下一轮内容无法优化
7. **不沉淀内容资产**: 合作结束素材散落 → 无法复用到详情页、私域和销售话术
