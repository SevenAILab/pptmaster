# OKR · 目标与关键结果

## 定义

OKR (Objectives and Key Results) 是目标管理工具,源自英特尔,用于把方向性目标 (O) 拆解为可衡量的关键结果 (KR)。模型卡强调: OKR 的价值是清晰目标、明确如何实现目标,并让各层级目标方向统一。

在品牌年度规划里,OKR 的作用是把"提升品牌势能""做强心智""拉动增长"这类愿望,转成可以执行、跟踪和复盘的年度目标体系。

## 适用场景

- **年度规划必检**: Sub-Agent ⑥ 用 OKR 定义年度品牌/营销目标
- **跨部门协同**: 让品牌、市场、渠道、产品、内容围绕同一目标对齐
- **策略落地追踪**: 将策略方向转为季度/月份可检查的关键结果
- **复盘与预算评估**: 判断投入是否推动了关键结果
- **不适合**: 替代 KPI 考核；OKR 更适合方向对齐和关键突破,不是所有日常指标清单

## 使用步骤

### 第 1 步 · 写 Objective

O 要有方向感和激励性,但不能空泛。它回答"今年最重要的品牌任务是什么"。

好 O 示例: "让植愈坊成为都市女性心中的情绪疗愈洁肤代表品牌。"

坏 O 示例: "提升品牌影响力。"

### 第 2 步 · 写 3-5 个 Key Results

KR 必须可衡量,并能证明 O 正在发生。不要把动作写成 KR。

| 类型 | KR 例子 |
|---|---|
| 心智 | 目标人群中"情绪疗愈洁肤"第一提及率达到 X% |
| 内容 | 小红书核心关键词自然搜索曝光达到 X |
| 转化 | 首购转化率从 X 提升到 Y |
| 留存 | 90 天复购率达到 X% |
| 资产 | 形成 3 个可复用品牌资产模板 |

### 第 3 步 · 分解到季度和团队

模型卡提醒下一级的 O 可以是上一级的 KR。品牌年度 OKR 要拆到季度主题和团队责任,否则停留在战略页。

### 第 4 步 · 匹配项目和资源

每个 KR 至少绑定 1-3 个关键项目、预算和负责人。例如"品牌出圈"对应达人内容、PR 事件、用户故事栏目。

### 第 5 步 · 设置复盘节奏

每月检查 KR 进度,每季度复盘 O 是否仍然成立。若外部环境变化,可调整项目,但不要频繁换 O。

## 输入输出示例

### 示例 1 · 植愈坊年度 OKR (247 书贯穿教学案例)

> **案例可追溯报告**: 植愈坊是 Seven 247 书全书贯穿的**教学性虚拟案例** (非真实商业体)。具体出处行号待 Codex 后续按 P0-4 模板填充, 当前仅标注教学属性。**禁止包装成"真实品牌成功案例"**。

**O**: 让植愈坊成为都市女性心中的情绪疗愈洁肤代表品牌。

| KR | 衡量方式 |
|---|---|
| KR1 | 小红书"情绪疗愈洁肤/疗愈手工皂"相关内容曝光达到 500 万 |
| KR2 | 首批 1 万名试用用户中,敏感肌安全满意度 ≥ 90% |
| KR3 | 私域 90 天复购率达到 35% |
| KR4 | 建立 3 个核心品牌资产: 口号、视觉锤、疗愈仪式内容模板 |

**关键洞察**: OKR 把"做品牌"拆成心智、体验、复购和资产四类可追踪结果。

### 示例 2 · SmallRig 年度 OKR

> **案例可追溯报告**: ⚠️ **待人工校对** — 本示例的 SmallRig 内容需逐字与 `assets/_raw/cases/标杆案例/smallrig/*.md` 真实页对照。如案例 PDF 中无对应内容, 必须按 P0-6 4 层 Fallback 改用 Tier 2 经典案例或 Tier 3 抽象示例, 禁止 LLM 编造 SmallRig 案例。

**O**: 从相机配件品牌升级为全球创作者模块化工具生态。

| KR | 衡量方式 |
|---|---|
| KR1 | 形成 5 条核心创作者工作流套装并完成全球上线 |
| KR2 | 新品首发适配周期缩短至 X 天以内 |
| KR3 | 全球创作者共创案例沉淀 100 个 |
| KR4 | 工具生态相关搜索与内容提及占品牌提及的 40% |

**关键洞察**: SmallRig 的 OKR 不只是卖更多配件,而是验证"工具生态"心智是否被市场接收。

## 常见误用

1. **O 太空**: "提升品牌力"没有方向 → 团队无法判断取舍
2. **KR 写成动作**: "发布 100 篇内容"是动作,不是结果
3. **KR 太多**: 十几个关键结果 → 没有关键
4. **只看销售**: 品牌年度 OKR 只写 GMV → 心智和资产无法被管理
5. **没有负责人和资源**: OKR 挂墙,项目无人推进
6. **不复盘**: 年初写完年末才看 → 失去管理价值
7. **把 OKR 当 KPI 惩罚工具**: 团队会保守填目标,失去突破性
