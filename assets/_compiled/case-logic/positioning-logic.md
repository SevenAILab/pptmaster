# 案例推导逻辑 · 品牌定位（源：贝比赋定位案，教学案例）

> 这是“怎么推导出一个定位”的思维范例，不是模板。学推理链，不学页数、版式、行业内容。

## 这类方案回答的核心问题

在已有竞争心智格局里，本品牌该抢占哪个“还空着、自己够得着、用户认”的心智位置？

## 推导链

1. 先承认现实，不急着喊定位：盘家底（基建、业务、运营）逼出核心问题，再给解决思路。定位是诊断后的结论，不是开场口号。
2. 四类证据交叉逼近：市场扫描（外部窗口）× 竞争态势（谁占了什么、哪有空位）× 消费者心智（脑中有哪些抽屉）× 自身优势（有什么可信资产）。每条都回扣“该占哪个位”。
3. 洞察支点是“心智抽屉有限”：定位不是“我想说什么”，而是“我能挤进哪个还空的格子”。
4. 收束成一句，再压成第一联想。定位要能被用户记住，也要能被销售、门店、传播反复复述。
5. 立刻给 RTB：定位提出后马上回答“凭什么信”。没有 RTB 的定位是空中楼阁。
6. 落地闭环：口号、VI、营销传播、产品/服务配称，把一句定位变成可执行体系。

## 典型页面推导方式

- 诊断页不要写“市场现状”，要写成判断：“增长正在从外部开店机会转向内部复购效率”。论据可以来自行业增长放缓、门店坪效、会员复购、竞品价格带变化。
- 竞争页不要写“竞品分析”，要写成判断：“主流竞品已经占据高质平价和匠人叙事两端，中间的日常专业空位仍未被清晰命名”。论据要说明谁占了什么、为什么用户会信。
- 用户页不要泛泛写画像，要回答“用户到底用哪个心智抽屉理解这个品牌”。如果用户已经有固定替代对象，定位必须重新定义替代关系；如果用户没有抽屉，定位必须先造品类解释。
- 自身资产页要区分“真的优势”和“自己以为的优势”。能被用户感知、被销售复述、被交付稳定证明的才是 RTB；内部流程、创始人偏好、模糊价值观不能直接当 RTB。
- 定位页是前面所有诊断的收束，不是凭空跳出来的一句 slogan。好的写法是“因为 A 竞争空位存在，因为 B 用户任务真实，因为 C 自身资产能证明，所以本品牌应该占据 X”。

## 反照搬检查

如果生成内容出现原案例行业名、原案例定位句、原案例章节顺序，说明模型在抄模板。正确做法是只继承推导动作：先诊断，后找空位，再压缩定位句，再给 RTB，最后落地。每个变量都必须替换成本客户自己的事实和判断。

## 迁移到新客户

- 学：诊断 → 找空位 → 收束一句 → RTB → 落地。
- 不学：原案例行业事实、原案例结论、页数、版式和章节命名。
- 新客户的定位必须从它自己的市场、竞争、用户、自身资产里重新长出来。
