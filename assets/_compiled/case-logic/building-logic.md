# 案例推导逻辑 · 品牌建设（源：童里建设案，教学案例）

> 学“怎么把定位落成一套能执行的品牌体系”的推理链，不学行业、页数、版式。

## 这类方案回答的核心问题

定位已定或方向大致清楚时，怎么把它翻译成产品、渠道、营销、服务都对得上的一整套可执行体系？

## 推导链

1. 先讲入局理由：从需求迁移、赛道断层、客户资产解释“为什么这个机会成立”，不是一上来做 VI。
2. 先诊断后建设：竞争、消费者、自身 SWOT 跑完，再进入品牌建设。
3. SWOT 不停在矩阵：把 S/W/O/T 转成 SO、ST、WO、WT 的具体动作。矩阵是手段，行动才是目的。
4. 核心是“定位之下的四大配称”：产品配称、渠道配称、营销配称、服务配称。这是建设案区别于定位案的关键。
5. 每个配称要有颗粒度：产品系列、价格、结构分开写；渠道有占比和角色；营销、服务落到具体资产。
6. 用阶段节奏把起步、卡位、扩张、迭代串成目标，让客户知道先做什么、后做什么、为什么。

## 典型页面推导方式

- 入局理由页先回答“为什么现在做建设是必要的”，而不是直接列品牌动作。可用需求变化、竞争窗口、自身资产成熟度来证明。
- 诊断页把竞争、用户、自身拆开，但每页都要回到“建设体系缺哪块”。不要把定位案的竞争空位分析原样重做一遍。
- SWOT/TOWS 页必须从矩阵走向行动：S-O 是先放大什么，S-T 是防守什么，W-O 是补齐什么，W-T 是暂时不做什么。
- 产品配称页要说明“定位如何进入产品结构”：旗舰、引流、利润、复购产品各自承担什么角色。
- 渠道配称页要说明“定位如何进入触点”：直营、经销、电商、私域、线下体验分别承担获客、转化、复购还是信任证明。
- 营销和服务配称页要给具体动作，不要停留在“打造内容矩阵”。动作要能被团队明天拆任务。

## 反照搬检查

建设案最常见的错误是把“品牌建设”写成一串传播动作。正确的结构是：定位或核心方向先成立，然后每一个业务系统都对齐它。若页面里没有产品、渠道、营销、服务的配称关系，只是在讲 logo、slogan、活动，那就不是建设案。

## 迁移到新客户

- 学：入局理由 → 诊断 → SWOT 转行动 → 四大配称 → 阶段节奏。
- 不学：原案例行业事实、具体配称内容、页数版式。
- 配称必须替换成本客户自己的产品、渠道、营销、服务。
