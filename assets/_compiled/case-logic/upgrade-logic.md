# 案例推导逻辑 · 品牌升级（源：SmallRig MI 升级案，真实标杆）

> 学“为什么要升级 + 升级成什么”的推理链。若引用 SmallRig 事实，必须回到原始资料核验。

## 这类方案回答的核心问题

旧定位为什么不够了？要升级到什么新定位，且这个升级在行业、用户、竞争和自身资产上都站得住？

## 推导链

1. 先立升级语境：说清旧身份的天花板和新机会的拉力。升级案必须先论证“为什么必须变”。
2. 行业趋势 + 用户洞察双支撑：用市场增长驱动、用户任务变化、效率需求或身份迁移，证明新方向有真实需求底座。
3. 痛点 → 收益 → RTB 对齐：每个新主张都要对应一个用户真实需求，以及品牌为什么能做到。
4. 竞争心智对标找新位：把旧参照系、新参照系、自身资产摆在一起，找到可以抢的新位置。
5. 用品牌层级承接升级：产品品牌、内容平台、子计划或服务体系构成层级，让“升级”有组织载体而非一句口号。
6. 收成新品牌核心：使命、愿景、主张、口号、人群、差异 RTB、产品系列一次立全。

## 典型页面推导方式

- 升级必要性页先写“旧身份为什么限制增长”。可以是用户任务变了、产品线变复杂、竞争者开始抢更高心智、原命名无法覆盖新业务。
- 趋势页不要孤立讲市场增长，要讲“趋势如何让旧定位失效”。增长只是背景，升级理由来自旧定位和新机会之间的张力。
- 用户痛点页要把痛点连接到新定位收益：用户为什么需要一个更完整、更高级或更可信的品牌角色。
- 竞争页要同时看旧参照系和新参照系。升级不是换一句更大的话，而是从一个竞争场搬到另一个竞争场，并证明自己站得住。
- 品牌层级页要承接复杂度：主品牌、产品线、内容平台、服务计划之间怎么分工，哪些保留，哪些改名，哪些新建。
- 新核心页要一次性回答“升级成什么”：使命、愿景、主张、人群、RTB、产品/服务体系都要咬合。

## 反照搬检查

升级案不能只写“更高端、更年轻、更国际化”。每一次升级都必须先证明旧身份的边界，再证明新身份的需求和可信理由。如果没有“为什么必须变”，只有“变成什么”，方案会像换皮。

## 迁移到新客户

- 学：升级语境 → 趋势和用户 → 痛点 RTB → 竞争找新位 → 品牌层级 → 新品牌核心。
- 不学：SmallRig 的影像行业事实、具体定位结论、页数版式。
- 必须先证明“本客户为什么必须升级”，再给“升级成什么”。
