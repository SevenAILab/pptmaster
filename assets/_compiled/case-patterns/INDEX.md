# Case Patterns · 案例 / SOP 结构样板

本目录把原始案例和 SOP 素材从 `assets/_raw/` 提炼成可自动注入 Sub-Agent prompt bundle 的结构样板。

## 已纳入来源

| Pattern | 原始来源 | 用途 |
|---|---|---|
| `brand-positioning-case-pattern.md` | `品牌定位案例.pptx` | 学习完整品牌定位案从 brief、诊断、研究、定位表达到传播落地的页序结构 |
| `brand-building-case-pattern.md` | `品牌建设案例.pptx` | 学习品牌建设案从赛道、SWOT、战略体系、四大配称到资产化落地的结构 |
| `smallrig-mi-upgrade-case-pattern.md` | `斯莫格MI升级案例.pdf` | 真实标杆案例事实源和 SmallRig 专属 P0-2 真实性边界 |
| `brand-management-sop-pattern.md` | `2024品牌管理全工作手册.pdf` | 将品牌战略、品牌表达、品牌沟通三段 SOP 作为全案结构和管理闭环参考 |

## 使用原则

1. Case Pattern 只提供结构、页序、证据路径和使用边界,不能替代客户真实输入。
2. 写 SmallRig 时必须回读 `_raw/cases/标杆案例/smallrig/page-NNN.md`,不能只引用本 pattern。
3. 写定位 / 建设 / 年度规划时,优先学习这些案例的“推导顺序”和“输出模块”,不要照搬虚拟品牌事实。
4. 所有引用到真实案例字段的输出必须保留 `data_refs.source`。
