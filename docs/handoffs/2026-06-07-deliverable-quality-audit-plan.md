# 交付质量审计 + 病因定位（Deliverable Quality Audit）

> 日期: 2026-06-07 ｜ 作者: Claude（战略计划）｜ 性质: 在继续建 Phase 7/8 之前的**质量校准关口**
> 缘起: Claude 实读了 Phase 3 已验收的 pptagent positioning 成品（80 页），发现引擎已稳过"红线/可追溯/方法论结构"，但**交付物本身有质量天花板**（主旨重复、单一观点摊薄、演绎多于实证、抽象高于可落地）。这类问题**任何红线测试都测不出来**，只能靠人读 + 对照实验定位。

## §1 核心判断（第一性原理）
- 已达成：**机器不作弊**（不编造、全分级、方法论注入、叙事连贯）。
- 未验证：**机器产出是否真的好**（一个创始人/咨询客户会不会觉得值钱）。
- 风险：在未回答"够不够好"前去做 Phase 7（PPTX 导出重构）/ Phase 8（Intake），是给单薄内容换精致外壳，**优先级错配**。
- 未知：pptagent 成品的"重复+演绎多"，是**引擎的病**还是**主体太薄（无市场数据）**？必须先定位病因，否则修错地方。

## §2 已观察到的质量缺陷（pptagent positioning，待验证是否普遍）
1. **主旨重复**：核心主张（AI PPT 工具→品牌策划方案 Agent）在 p1/p2/p4/p5/p7/p11/p12 反复重述。
2. **同质页**：p8/p9/p10 三张近乎相同的 SWOT 2×2，仅换层级标签。
3. **演绎>实证**：多为"应如何定位"的逻辑推演，少"调研发现具体数字/真实用户原话"的实证。
4. **抽象>可落地**：战略话术多，缺可执行动作、量化锚点、真实用户证据。

## §3 计划：三步走，用对照实验定位病因

### Step 1 — 收尾 building（在途，~1 chunk）
- Phase 6 building 续跑跑完（13/15→15/15）、assemble+render、过 CP-6 A–F。**这步只是把在飞的收掉，不阻塞 Step 2 的战略判断。**

### Step 2 — 跑一本「证据充足主体」做对照（关键实验）
- 选 **smallrig**（真实品牌、有行业/竞品/人群真实可查数据；inputs 已存在）跑 positioning 全卷真跑（当前引擎，新建 run-id）。
- 成本上限 $8、墙钟 45 分钟、env unset 自检、断点续跑护栏，沿用既有真跑手册。
- 目的：**控制变量**——同一引擎、换成有数据的主体，看输出是否变"实"（实证页增多、重复下降、可落地增强）。

### Step 3 — 结构化质量审计（pptagent-薄 vs smallrig-厚 对照）
对两本成品按**咨询方案质量维度**逐项打分（人读 + 可机检指标结合）：
| 维度 | 机检信号（可量化） | 人读判断 |
|---|---|---|
| 洞察密度 | 不同 action_title 的语义去重率；每页是否引入新信息 | 每页是否有"新的一刀"，还是重复上页 |
| 主旨重复 | 核心主张关键词在多少页重复出现 | 是否同一结论换皮多遍 |
| 实证 vs 演绎 | evidenced 页里"带真实外部数字/用户原话"的比例 | 结论是"查到的"还是"想出来的" |
| 可落地性 | 含具体动作/数字/时间的页占比 | 创始人能否照着做 |
| 叙事递进 | 章节间是否层层推进（非平铺重述） | 读完是否有"被层层说服"感 |

**产出判定（决定下一阶段往哪走）：**
- **分支 A — 若 smallrig（有数据）明显更实更好** → 天花板是"主体太薄"，引擎没大病。下一步：① 加「证据成熟度门」——主体太薄时显式提示"探索性方案，建议补一手资料"，并主动收窄页数（薄主体不强行摊到 80 页）；② 然后才进 Phase 7（导出）/ Phase 8（Intake）。
- **分支 B — 若 smallrig 同样重复/演绎多** → 天花板是"引擎的病"。下一步开「洞察密度/反冗余」内容工程 phase：跨页主旨去重、强制每页一个不同洞察、页数预算纪律（与其 80 页注水，不如 40 页页页有料）、写手从"复述定位"转向"推进新证据"。

## §4 为什么先做这个，而不是 Phase 7/8
- Phase 7（PPTX 导出重构）是把"文件格式"做漂亮——但若内容有质量天花板，等于精装单薄内容，ROI 低。
- Phase 8（Intake 一句话需求）是入口体验——但产出质量未达标前，入口再顺也只是更快产出"还不够好"的方案。
- **先把"产出够不够好 + 病在哪"搞清，再决定把工程力气投向哪**——这才是最短路径。

## §5 红线 & 护栏（不变）
- Step 2 真跑沿用全部既有红线与防空转/成本闸。
- 质量审计**不放宽任何红线**；它评的是"在不作弊前提下，方案好不好"，是红线之上的更高一层。

## 给小白的讲解
- **我发现了什么:** 我把那本已验收的 pptagent 方案**真的逐页读了**。好消息：它确实不编数据、结构专业。但说实话——**它有点"车轱辘话"**：同一个核心观点（从"AI PPT 工具"改叫"品牌策划方案 Agent"）翻来覆去说了十几页，还有三页几乎一模一样的表格只换了标题。作为给客户的方案，这是"诚实但不够精彩"。
- **但这可能不全怪引擎:** pptagent 是个没什么市场数据的小项目，没数据就只能"讲道理"，自然容易重复。所以现在不能急着下结论说"引擎不行"。
- **我建议怎么办（先别急着建新功能）:** 拿一个**有真实数据的品牌（smallrig）**用同一个引擎再跑一本，两本对比着看——如果有数据的那本明显更扎实，说明是"原料薄"的问题；如果两本都车轱辘话，那才是引擎要改。**先做这个对照实验，再决定下一步把力气花哪。**
- **为什么不先做之前说的"换 PPT 导出工具":** 因为那是把"文件外观"做漂亮；可如果内容本身还不够精彩，等于给一幅普通的画配个金画框，不划算。**先确认内容够不够好，最重要。**
