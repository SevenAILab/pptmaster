# pptmaster 引擎重构设计 · 4 道工序流水线（Stage 1：先做引擎）

- 日期：2026-05-29
- 作者：Claude（设计）/ Seven（决策）/ Codex（后续执行）
- 状态：设计稿，待 Seven 评审 → 通过后转「实现计划」
- 适用范围：本期（Stage 1）只设计「引擎大脑」，SaaS 外壳（界面/登录/付费/托管）推迟到 Stage 2/3

---

## 0. 一页速览（TL;DR）

把现有 pptmaster 进化成一条 **4 道工序的流水线**，让"一句模糊需求"稳定产出"**咨询级、可在 PowerPoint 里编辑**"的品牌/营销策略方案：

```
模糊需求
  → ① 入口追问（新建）     带 A/B/C 选项一步步问清，产出 表单+摘要+根问题+研究brief
  → ② 深度调研（保留+护栏） 6 子代理联网查证、带出处写，事实可追溯
  → ③ 咨询级成稿（升级）    咨询纪律 + 改成可转 PPT 的 .S/1080×608 HTML
  → ④ 转可编辑 PPT（新增）  接 html2ppt(dom-to-pptx)，出可编辑 .pptx
可编辑的咨询级方案
```

**本期只做"大脑"**：四道工序能在「与 Claude 对话 / 命令行」里跑通、能逐步验收。
**本期不做**：报告推荐的重型架构（FastAPI / LangGraph / Docker 沙箱 / 向量数据库 / React）；SaaS 外壳。
**红线**：失败必报错，不静默兜底、不伪造数据；每个数字可追溯（来源+日期+等级）。

## 1. 背景与目标

### 现状（已有，保留）
- 总策划师（Chief Strategist）+ 6 个子代理：行业分析 / 竞品分析 / 消费者洞察 / 品牌定位 / 品牌打法 / 年度规划。
- 已有：蓝图（blueprint）组装、瑞士版式渲染器（`smart_layout` → S03…S22 + `template-swiss.html`）、来源分级（Codex 刚修：T1 一手 / T2 权威二手 / T3 媒体 / T4 UGC）。
- 资产层：Seven 品牌方法论书、营销模型卡、标杆案例。

### 缺口（本设计要补）
1. **没有交互式入口追问**：现有入口是"填 10 字段表单 + 手写摘要 + 一次性套战略问题模板"（`strategic-question.mjs` 是静态生成器，不是一步步追问）。
2. **渲染输出是"网页 viewer"不是"可转 PPT 规范"**：现有输出是 `#deck`/`#nav`/`#overview` 结构，不是 `.S`/1080×608 自包含规范。
3. **没有"转可编辑 PPT"这一步**：现在只交付 HTML，客户拿不到 `.pptx`。

### 目标（Stage 1）
让一句模糊需求经 4 道工序，稳定产出一份**咨询级、可在 PowerPoint 里二次编辑**的品牌/营销策略方案；且**每一步可被非技术用户验收**、**失败必报错**。

## 3. 总体架构：4 道工序流水线

| 工序 | 现状 | 改造 | 改动量 |
|---|---|---|---|
| **① 入口追问** | 填表单+手写摘要+一次性套战略问题 | 🆕新建：带 A/B/C 选项一步步追问，自动产出 表单+摘要+根问题+研究 brief | 大 |
| **② 深度调研** | 6 子代理已能跑，来源分级刚修好 | ✅保留为主：加引用纪律护栏 | 小 |
| **③ 咨询级成稿** | 瑞士版式渲染器+模板，输出网页 viewer | ⬆️升级：咨询纪律 + 改成可转 PPT 的 `.S` 规范 | 中 |
| **④ 转可编辑 PPT** | 没有这一步 | 🆕新增：接 html2ppt（dom-to-pptx） | 中 |

**接缝原则**：各工序之间用"明确的文件/接口"交接（见 §5）。每个接缝都是一个可单独验收的检查点；将来真要升级某一环（如换更强编排、加向量检索），可替换该环而不动其余。

## 4. 工序详细设计

### ① 入口追问（新建）

**目的**：把非技术用户的模糊需求，用一步步带选项的提问，问到"意图清晰、证据足够"，产出结构化输入。

**4.1 五个品牌维度（替换原技能的 G/S/C/R/E）**

| 维度 | 含义 | 0 分 | 1 分 | 2 分 |
|---|---|---|---|---|
| 主体 | 品牌/产品是谁、定位 | 只有名字 | 有模糊定位 | 定位+差异点清晰 |
| 人群 | 目标人群、动机、放弃点 | 说不清 | 笼统画像 | 具体场景+最近一次真实行为 |
| 竞争 | 竞品、对标、差异 | 不知道 | 列得出名字 | 知道竞品最好/最差点 |
| 目标 | 这次方案要解决什么、主指标 | 不明确 | 有方向 | 有可衡量目标 |
| 证据 | 支撑判断的事实/数据 | 纯主观"我觉得" | 间接（竞品都这么做） | 直接（上月 30% 投诉是关于 X） |

> 证据维 0/1/2 的锚点直接借自 `router-logic.md` 的 E 维定义，几乎不改。

**4.2 成熟度与红线（硬阈值）**
- 总分 → 🔴1-3 萌芽 / 🟡4-6 成长 / 🟢7-8 成熟 / ✅9-10 就绪。
- **红线/硬阈值（直接复用追问技能的 Hard Threshold Override）**：**证据维=0 时，不准出"确认结论"，强制降级**（标注 `Capped: 证据=0, 需补证据`），并触发三选一：① 我帮你抓权威二手资料 ② 你上传一份现有材料 ③ 先按通用框架出"草稿（待确认）"。**绝不编数据填空。**

**4.3 路由（自上而下优先级，借自 `router-logic.md` R0-R7，映射到品牌维）**
1. 有冲突（如"高端定位 vs 大众人群"）→ 先拆冲突。
2. 高分且各维≥1 且无冲突 → 快速结构化，直接出稿。
3. 某维=0 → 优先补该维。
4. 证据=0 → 走"证据补缺"路径（触发抓取/上传）。

**4.4 追问母模板（逐字保留骨架，借自 `question-bank.md` L193-215）**
```
本轮理解：<一句话回显，降低焦虑>
P0（请优先回答）
1. <问题> —— 为什么问：<一句话理由>
   💡 参考答案：
   A. <具体到带数字/场景的示范答案>
   B. <…>
   C. <…>
2. <问题> —— 为什么问：<…>
P1（选答）
4. <问题>
可直接回复：`1A 2C 3是`，也可自由补充。
```
**节奏护栏**：每轮抽 1-3 个 P0 + 0-2 个 P1（**单轮不把一条路径问完**）；每轮结尾更新「已确认 / 假设 / 待确认」；**最多 3 轮**。

**4.5 "我不知道"处理（借自 `router-logic.md` L251-308）**
- 连续答不上来计数（跨维累计）：到第 2 次 → 转"温柔问法"（Probe Mode，例：把"目标用户是谁"改成"你脑海里第一个用户是谁？"）；到第 3 次 → "提前收尾"（Early Pivot）：停止追问，输出"需求探索摘要（未就绪）"+ 三条出路（抓二手 / 上传 / 通用草稿）。

**4.6 输出（对接现有结构）**
- `inputs/{client_slug}/form.json`（沿用现有 10 字段）
- `inputs/{client_slug}/summary.md`
- `inputs/{client_slug}/strategic-question.md`（根问题）
- **研究 brief**：借 OpenAI 的 prompt-rewriter 思路，把模糊需求扩写成含"范围/对比/指标/地区/期望输出"的完整 brief，交给 ② 的每个子代理。

**4.7 本期形态**：通过"与 Claude 对话 / 命令行"体验，**不做网页界面**（网页界面是 Stage 2）。

### ② 深度调研（保留 + 护栏）

**目的**：6 子代理联网查证、带出处写，事实可追溯。

**保留**：总策划师 + 6 子代理 + 联网边界（行业/竞品必须查；洞察/年规可选；定位/打法不直接查）+ 来源分级（T1-T4，Codex 刚修）。

**护栏 1 — 派活"四要素合同"（借自 Anthropic 编排器-工人）**：每个子代理任务包必含 `①目标 ②输出格式 ③信源指引 ④边界`。配"用力档位"：简单查证 = 查 3-10 次；对比 = 2-4 个子代理各 10-15 次。在 prompt 里加"信源质量启发式"（优先一手/权威源，避开 SEO 内容农场）。

**护栏 2 — 引用纪律（借自 CitationAgent + Gemini MIG）**：
- 每个数字**句尾强制挂**〔来源 id + 日期 + 等级 T1-T4〕。
- 子代理产出后**单独跑一遍"引用校验"**：无出处的数字一律打回到 `assumptions`，不得伪装成事实。
- 生成 4 列**证据台账**：`来源 / 日期 / 偏见判断 / 关键发现`（也供 ③④ 调用）。

**护栏 3 — 重规划红线（借自 Claude DR）**：可信度 < 60% 或矛盾信息 > 30% → 重新规划（防被单一营销软文带偏）。

**系统提示词骨架（可直接抄那份合成 XML 模板，`项目借鉴方案.md` L90-122，改成品牌语境）**：`角色=带科学家心态的策略分析师（查证/追溯证据链/识别偏见/中立）→ 规划（拆 3-5 子问题）→ 执行约束 → 报告标准`。

**工程旋钮（轻量）**：广度+深度参数控制查多深、最大并发（默认 5）、按角色用不同档模型省钱。

**反面教材警示**：借结构**不借内容**——某开源模板（VoltAgent）示例输出里有"调研 2400 个消费者 / 市场 42 亿"这类**编造数字**，严禁进入我们的产出（红线）。

### ③ 咨询级成稿（升级）

**目的**：把结论渲染成"咨询级 + 可转 PPT"的 HTML。两个半边。

**半边 A — 内容纪律（搬咨询技能）**
- **每页 4 槽位强制结构**（借 `consulting-exhibit-system.md` L9-41）：`行动标题(=结论) / 主证据对象 / 含义建议 / 出处置信度`；**缺槽位 → 报错**。
- **行动标题**：是结论不是话题标签；"只读标题能讲完整个故事"（SCR/SCQA）。
- **框架库优先级**（借 `consulting-frameworks.md`）：SCR/SCQA（全局）> 金字塔（每页结论先行）> 2x2 > Before/After > MECE。
- **量化置信度 A/B/C**（借 `value-kpi-library.md`）：无 A 级证据（基线+公式+来源）**不准写精确数**（如 8.6%）；KPI 卡 6 行：`指标/基线/目标/来源/负责人/置信度`。
- **中文营销语气黑名单**（借 `consulting-frameworks.md` L99-118）：禁用"赋能 / 打造闭环 / 构建生态 / 标志着 / 里程碑 / 业内认为"等空话。

**半边 B — 格式规范（对齐可转 PPT）**
- 每页 `.S` 容器，固定 **1080×608**；**自包含**（无外链字体/图片/CDN）；所有文字是**真文本**（不进图片/canvas/复杂 SVG）。
- **表格语义双值**（按"将来怎么编辑"选，不按视觉）：要逐行列编辑的 → `<table data-pptx-role="native-table">`；视觉矩阵（2x2/热力图）→ div/grid + `data-pptx-role="shape-matrix"`。
- **设计系统确切数值**（抄 `design-system.md`）：主标题 22-26px；卡片标题 13-15px（13 硬下限）；正文 11-12px；**硬地板 9px**（页脚 8.5px 例外）。
- **反空心铁律**（`design-system.md` L68-76）：**禁止用 inline `style="font-size:"` 把字缩到小于 CSS 类默认值**（这是"空心页"头号成因）；要小字就建专用类。
- **品牌营销优先版式 7 个**（借 `slide-patterns.md`）：开篇结论 / 现状诊断 / 市场盘+受众细分 / 竞争定位 2x2 / 策略选择+优先级 / 价值论证 / 落地节奏。

**技术选择（已拍板）**：直接**改造现有渲染器输出 `.S` 规范**（单一来源、长期干净），而非"再加一层把网页壳转成 PPT 壳"的转换器。

**迁移注意（重要坑）**：咨询技能整体是"制造业味"的（KPI 库/评分维度/架构图全是工厂语境）。**框架骨架与设计数值可直接抄，但内容词表与评分维度必须换成品牌营销语境**，否则会输出"工厂味"的营销稿。

### ④ 转可编辑 PPT（新增）

**目的**：把 ③ 产出的 `.S` HTML 转成可编辑 `.pptx`。

- **把现有 html2ppt 搬进 pptmaster**当流水线一步（自包含、可复现），而非依赖 `Downloads` 的外部文件夹。html2ppt 底层依赖 `@halobiron/dom-to-pptx`。
  - 旁证：报告**独立印证**了"DOM → 原生 PPTX 形状/文本框"这条路线正确（并点名了 `atharva9167j/dom-to-pptx`、`PptxGenJS` 等同类实现）。它们说明"文字框保留字号/字体/颜色、HTML 表格转可编辑 PPT 表"是真能做到的。
- 一条命令出 3 个版本 + 1 份报告：`推荐可编辑版`（默认交付）/ `最大可编辑版` / `原始保真版` + `转换报告.md`。
- **护栏**：① 转换失败必须报错（红线，不静默兜底）；② 校验报告里"原生 PowerPoint 表格数" ≥ 设计的真表格数，偏低即说明 ③ 用了假表格，打回。
- **可选加分（后续）**：引入 PPTAgent 式"内容/设计/连贯"三维质检做成稿后自动打分。

## 5. 数据流与接口（干净接缝）

```
模糊需求
  ─①─▶ inputs/{slug}/form.json + summary.md + strategic-question.md + research-briefs
  ─②─▶ 6 子代理产出 + evidence-ledger（证据台账）
  ─蓝图组装─▶ deck.json（含 smart_layout + 各页 4 槽位内容 + 出处）
  ─③─▶ deck.html（.S / 1080×608 / 自包含 / 真文字 / 真表格）
  ─④─▶ *_推荐可编辑版.pptx 等 + 转换报告.md
```
每个箭头是一个**明确的文件/接口**；任一环缺失或不合规 → **报错停下**。

---

## 6. 错误处理与红线

- **失败必抛错**，不静默兜底、不伪造数据。
- **证据不足 → 触发抓取/上传/草稿**，绝不编数据填空。
- **每个数字可追溯**（来源 + 日期 + 等级 T1-T4）。
- **借鉴的开源模板里的"示例假数字"严禁进入产出**。
- ③ 出稿前的质检关、④ 的转换报告校验，任一不过 → 打回，不放行。

## 7. 验收与测试（怎么知道做对了）

每道工序都能**单独跑、单独验收**（命令行 / 与 Claude 对话），非技术也能查：

| 工序 | 自动门禁 | Seven 怎么自己查 |
|---|---|---|
| ① | 证据=0 时必须触发抓取/上传，不准硬出稿 | 打开 `inputs/{slug}/` 看三件套齐不齐、读着对不对 |
| ② | 引用校验 pass；证据台账存在 | 随便翻一份子代理产出，盯任一数字有没有〔来源+日期〕 |
| ③ | 7 维评分 ≥ 80 分线 + 9 道质检关 | 只读大标题能否讲完故事；浏览器看页面无空心、无黑名单词 |
| ④ | 转换不报错；原生表格数对得上 | 打开"推荐可编辑版.pptx"点标题/表格能改字 |
